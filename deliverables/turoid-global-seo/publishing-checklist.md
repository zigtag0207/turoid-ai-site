# Publishing checklist — Global “family office software” cluster

## Before publish

- Create pages with slugs exactly as drafted (or adjust consistently) and ensure each page uses a **self-canonical** URL.
- Add these pages to `sitemap.xml` (Turoid already serves `https://www.turoid.ai/sitemap.xml`; include new URLs once published).
- Ensure `robots.txt` continues to allow crawling and references the sitemap.
- Add the schema JSON-LD blocks (one per page) via the site template or schema manager.
- Ensure `og:image` uses absolute URLs everywhere (homepage currently uses a relative path).

## Internal linking (minimum)

- Add a navigation link (or homepage “Solutions” link) to `/family-office-software/`.
- Add a “Core capabilities” section on `/family-office-software/` linking to the four capability pages.
- Add “Back to Family Office Software” links on each capability page.
- Add HK regional links:
  - Hub page: include `/family-office-software-hong-kong/`
  - HK pages: add a “Global overview” link to `/family-office-software/`

## After publish (week 1)

- Submit the new URLs in Google Search Console URL Inspection (request indexing).
- Watch indexing coverage and “Discovered — currently not indexed”.
- Track whether `/family-office-software/` begins to surface for “family office software” and “family office software comparison”.
